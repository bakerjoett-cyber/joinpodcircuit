How to Go Live — No Developer Needed

Your site is fully built. Follow these steps in order and PodCircuit will be live on the internet with a real database within about 30 minutes. Everything here is free or cheap, and none of it requires coding.

1 Set Up Your Database (Supabase) FREE

Supabase is a free database that stores your guest profiles in the cloud so anyone can sign up from any device. You just need to create an account and copy two values into your site files.

1
Go to supabase.com and create a free account
Click "Start for free" — sign up with Google or email. It's completely free for small projects.
2
Create a new project
Click "New Project" → give it a name like "podcircuit" → set a database password (save this somewhere) → click "Create new project." Wait about 60 seconds for it to set up.
3
Create the guest_profiles table
In your project, click "Table Editor" in the left sidebar → "New Table" → name it guest_profiles → enable "Row Level Security" → click Save. Then run this SQL to create all the columns:
-- Run this in Supabase: SQL Editor → New Query → paste → Run create table guest_profiles ( id uuid default gen_random_uuid() primary key, user_id text unique not null, email text not null, first_name text, last_name text, title text, city text, website text, story text, turning_pt text, achieve text, ig_handle text, ig_followers integer default 0, tt_handle text, tt_followers integer default 0, yt_url text, yt_subs integer default 0, tw_handle text, tw_followers integer default 0, li_url text, li_followers integer default 0, categories text[] default '{}', points text[] default '{}', availability text default 'available', format text default 'both', pod_exp text default 'never', clip text, photo_url text, status text default 'draft', submitted_at timestamptz, created_at timestamptz default now(), updated_at timestamptz default now() ); -- Allow anyone to read approved profiles (for the browse page) create policy "Public can read approved profiles" on guest_profiles for select using (status = 'approved'); -- Allow users to manage their own profile create policy "Users manage own profile" on guest_profiles for all using (true) with check (true);
4
Get your two keys
Click "Project Settings" (gear icon) → "API" → you'll see two values: Project URL and anon public key. Copy both — you'll need them in the next step.
5
Paste your keys into the config file
Open supabase-config.js (included with your files) and replace the two placeholder values with your real keys. That's it — your database is connected.
That's the only "technical" step — and it's really just copying and pasting two lines of text. No coding required.
2 Set Up Email Notifications (Formspree) FREE

Every time someone submits a profile, you'll get an email. Takes 5 minutes.

1
Go to formspree.io and create a free account
Sign up free — you get 50 submissions/month at no cost.
2
Create a new form
Click "New Form" → name it "PodCircuit Guest Applications" → set your email address as the recipient → click Create.
3
Copy your Form ID and paste it in supabase-config.js
Your form will have a URL like formspree.io/f/abcd1234 — the part after /f/ is your ID. Open supabase-config.js and paste it where it says FORMSPREE_ID.
3 Buy Your Domain Name ~$10/year

Your domain is your website address — like podcircuit.com. Here's how to get one in under 10 minutes.

1
Go to namecheap.com
Search for the domain name you want. Try: podcircuit.com, getpodcircuit.com, joinpodcircuit.com, podguestlist.com — whatever fits your brand.
2
Buy it
Most .com domains cost $9–$14/year. Add to cart and check out. Skip the extras — you don't need them.
Don't overthink the name. You can always redirect a domain later. Get something simple and move on — the product matters more than the URL.
4 Host Your Site (Netlify) FREE

Netlify puts your files on the internet. It's free, takes about 5 minutes, and has a drag-and-drop upload.

1
Go to netlify.com and create a free account
Sign up with email or Google — completely free.
2
Drag and drop your files
On your Netlify dashboard, there's a box that says "drag and drop your site folder here." Put all 5 of your files in one folder on your computer and drag the whole folder in. Your site will be live instantly at a temporary URL like random-name.netlify.app.
3
Connect your custom domain
In Netlify: Site Settings → Domain Management → Add Custom Domain → type your domain → follow the steps to point your Namecheap domain to Netlify. Netlify walks you through it step by step. Takes about 10 minutes and then your site is live at your real URL.

✅ At this point your site is fully live

Anyone in the world can visit your URL, create a guest profile, and podcast producers can browse and filter guests. You manage approvals from the admin dashboard.

6 Your 5 Files — What Each One Does
1
podcast-circuit.html — The main browse page
What podcast producers see. Shows all approved guest profiles with search and filters.
2
podcircuit-profile.html — Guest profile builder
Where guests create an account, build their profile, and submit for review.
3
podcircuit-admin.html — Your admin dashboard
Password-protected. Where you approve or reject guest submissions. Approved guests go live instantly.
4
podcircuit-apply.html — Alternative application form
A multi-step form option for guests who prefer a guided experience instead of the self-build profile page.
5
supabase-config.js — Your database connection
The one file you need to edit. Paste your Supabase URL and key here. All other files pull from this one so you only have to do it once.
